# git

# 下载和安装

下载地址：https://www.git-scm.com/download/

安装时一直下一步即可

git仓库有github以及gitee码云

# 配置用户名和邮箱

  1. 配置身份，设置用户名与邮箱
git config --global user.name "username"
//username为git的用户名

git config --global user.email "your email"
//your email为git上的邮箱
  1. 查看配置的用户名和邮箱（和设置时差不多，去掉用户名和邮箱即可）
git config --global user.name

git config --global user.email

# 创建代码仓库git init

用于保存版本管理所需要信息，所有本地提交的代码都会被提交到这里，如果有需要还可以在推送到远程仓库中

1.创建本地仓库目录

2.git init （会在指定的目录下生成一个隐藏的.git文件夹）

# 命名总结

# 最常用命令

​ 添加更改 --》提交到本地仓库 --》推送到远程

​ git add 文件路径 （git add . 为提交所有文件）

​ git commit -m "提交了vue.js文件"

# 其他一级常用命令

​ 清空当前页数据： clear

​ 当log太多时，按回车键换页显示

​ 当log太多时，正确的退出log显示界面： q-->回车

# 代码回溯相关命令

​ 检出当前版本最初始代码： git checkout .

​ 回溯到上一个版本： git reset --hard HEAD^^^有几个^符号就会回溯几次版本

​ 回溯到指定版本号的版本：git reset --hard 版本号前七位

​ 查看你的所有版本记录,包括回溯了的版本 git reflog

# git分支

  1. 查看当前的版本库中有哪些分支： git branch -a

  2. 创建一个分支：git branch 分支名 如：git branch version2

  3. 若分支前有一个*号，表示当前分支为我们代码所在分支

  4. 切换当前分支：git checkout 分支名 如： git checkout version2

  5. 在分支A修改并且提交的代码并不会影响到分支B

  6. 合并分支内容：git merge branchB 这样就会把分支B内容合并到当前分支

  7. 删除分支branchA：git branch -D branchA

  8. 从远程克隆时(git clone 地址)，只能克隆主分支。除非加上 -b 分支名（git clone -b dev 地址）

  9. 当进行切换分支时，你的本地代码也会相应的进行切换

  10. 如果在分支A上进行了更改，然后却没有提交，那么在切换到分支B上时，会和分支B进行合并！

创建一个分支后，要提交的话，需要先对现有内容进行一定的更改才行。
