# JavaScript

# 数据类型

# 基本数据类型

Number数值型、Boolean布尔类型、String字符串、Undefined未定义、Null空指针、Symbol

# 1.Number类型

​ Number类型包含整数和浮点数（浮点数数值必须包含一个小数点，且小数点后面至少有一位数字）两种值。

​ NaN:非数字类型。特点：① 涉及到的 任何关于NaN的操作，都会返回NaN ② NaN不等于自身。

​ isNaN() 函数用于检查其参数是否是非数字值。

​ isNaN(123) //false isNaN("hello") //true

# 2.String类型

​ 字符串有length属性。

​ 字符串转换：转型函数String(),适用于任何数据类型（null,undefined 转换后为null和undefined）；toString()方法（null,defined没有toString()方法）。

# 3.Boolean类型

​ 该类型只有两个值，true和false

# 4.Undefined类型

​ 只有一个值，即undefined值。使用var声明了变量，但未给变量初始化值，那么这个变量的值就是undefined。

# 5.Null类型

​ null类型被看做空对象指针，前文说到null类型也是空的对象引用。

# 引用数据类型

Object对象、Array数组、Function函数

# typeof 操作符

由于js中的变量是松散类型的，所以它提供了一种检测当前变量的数据类型的方法，也就是typeof关键字

typeof  123　　 //Number

typeof  'abc'　　//String

typeof   true    //Boolean

typeof   undefined  //Undefined

typeof   null     //Object

typeof   { }      //Object

typeof   [ ]      //Object

typeof   console.log()    //Function

null类型进行typeof操作符后，结果是object，原因在于，null类型被当做一个空对象引用。

# constructor 属性

constructor 属性返回所有 JavaScript 变量的构造函数。

"John".constructor                 // 返回函数 String()  { [native code] }
(3.14).constructor                 // 返回函数 Number()  { [native code] }
false.constructor                  // 返回函数 Boolean() { [native code] }
[1,2,3,4].constructor              // 返回函数 Array()   { [native code] }
{name:'John', age:34}.constructor  // 返回函数 Object()  { [native code] }
new Date().constructor             // 返回函数 Date()    { [native code] }
function () {}.constructor         // 返回函数 Function(){ [native code] }

//判断类型
myArray.constructor.toString().indexOf("Array") > -1;

myDate.constructor.toString().indexOf("Date") > -1;

# 变量作用域

全局变量和局部变量

函数内部可以直接读取全局变量，但在函数外部无法读取函数内部的局部变量

注意点：在函数内部声明变量的时候，一定要使用var定义变量。不用的话，声明的是一个全局变量

# 全局作用域

如果一个变量在函数外面或者大括号{}外声明，那么定义的一定是一个全局作用域的变量。

const globalVariable = 'some value'

声明了全局变量，那就可以在任何地方都可以使用它，包括在函数中也可以使用。

const hello = 'Hello CSS-Tricks Reader!'

function sayHello () {
  console.log(hello)
}

console.log(hello) // 'Hello CSS-Tricks Reader!'
sayHello() // 'Hello CSS-Tricks Reader!'

但，全局变量会造成冲突或污染

比如你使用const或者let声明变量，那么在命名冲突的时候会报错

let thing = 'something'
let thing = 'something else' // Error, thing has already been declared

若你使用var声明变量，第二个声明的同名变量就会覆盖前一个。

var thing = 'something'
var thing = 'something else' // perhaps somewhere totally different in your code
console.log(thing) // 'something else'

# 局部作用域

javaScript中一般就包括俩种局部作用域：函数作用域和块作用域

# 函数作用域

当你在函数中声明一个变量，就只能在这个函数范围内使用。外部是使用不了这个变量的

function sayHello () {
  const hello = 'Hello CSS-Tricks Reader!'
  console.log(hello)
}

sayHello() // 'Hello CSS-Tricks Reader!'
console.log(hello) // Error, hello is not defined
# 块作用域

不在函数中时，当你在大括号{}中使用const或者let声明的变量，那么这个变量就只能在这个大括号{}的范围内使用

{
  const hello = 'Hello CSS-Tricks Reader!'
  console.log(hello) // 'Hello CSS-Tricks Reader!'
}

console.log(hello) // Error, hello is not defined

平常块作用域都是被我们作为函数作用域的一个子集

# 函数提升和作用域

当你申明一个函数时，它总会提升到作用域顶部。这俩种结果相同

// This is the same as the one below

  console.log('Hello CSS-Tricks Reader!')
}

// This is the same as the code above

  console.log('Hello CSS-Tricks Reader!')
}

当声明一个函数表达式，函数就不会提升到作用域顶部

const sayHello = function () {
  console.log('aFunction')

# 函数不能相互调用各自的作用域

当你定义函数时，他们不能够相互使用各自的作用域，虽然可以互相调用

function first () {
	const firstFunctionVariable = `I'm part of first`
}

function second () {
  first()
  console.log(firstFunctionVariable) // Error, firstFunctionVariable is not defined
}

# 嵌套作用域

当一个函数在另一个函数内定义，内部的函数能够访问外部函数的变量。称为词法作用域，

但外部函数依旧不能访问内部函数的变量。

function outerFunction () {
  const outer = `I'm the outer function!`

  function innerFunction() {
    const inner = `I'm the inner function!`
    console.log(outer) // I'm the outer function!
  }
  innerFunction()
  console.log(inner) // Error, inner is not defined
}
outerFunction()

当你想要内部函数的元素就可以使用return

function outerFunction () {
  const outer = `I'm the outer function!`

  function innerFunction() {
    const inner = `I'm the inner function!`
    console.log(outer); // I'm the outer function!
    return inner
  }
  console.log(innerFunction()) // I'm the inner function!
}
outerFunction()

这作用域就像一面单面镜。你能够看见外面的人，而外面的人无法看见你

嵌套作用域就可以理解为多层单面镜

# 闭包

当你在一个函数内部创建一个函数时，就创建了一个闭包。内部函数就是闭包，闭包总是会return出来，所以稍后可以使用外部函数中的变量。

function outerFunction () {
  const outer = `I see the outer variable!`

  function innerFunction() {
    console.log(outer)
  }

  return innerFunction
}

outerFunction()() // I see the outer variable!

注：outerFunction调用后得到的是innerFunction函数，且innerFunction函数没有自调用，所以需要俩个()()

当内部函数需要return时，你可以直接return函数声明，代码量上会有一定的减少。

function outerFunction () {
  const outer = `I see the outer variable!`
  return function innerFunction() {
    console.log(outer)
  }
}
outerFunction()() //I see the outer variable!

function outerFunction () {

  2. 创建私有变量

# 巧用闭包解决问题

当你从一个函数中返回一个值时会产生副作用。很多事情都会有副作用，比如 Ajax 请求，timeout 或者一个 console.log。

function (x) {
  console.log('A console.log is a side effect!')
}

当你使用闭包来解决副作用时，你通常会关心这样弄乱你代码，像 Ajax 或者 timeouts。

让我们通过一个例子来理清这些。

比如你想要为你给你朋友的生日制作一个蛋糕。这个蛋糕需要一秒钟制作完成，所以你写了一个函数，在一秒后打印出 made a cake。

function makeCake() {
  setTimeout(_ => console.log(`Made a cake`), 1000)
}

正如你所看见的，这个“制作蛋糕”的函数有一个副作用：延迟。

更进一步，你想要你的朋友选择一个蛋糕口味，你能加一个口味到你的 makeCake 函数。

function makeCake(flavor) {
  setTimeout(_ => console.log(`Made a ${flavor} cake!`), 1000)
}

当你运行这个函数时，提示蛋糕在一秒钟后立即制成。

makeCake('banana') // Made a banana cake!

这个问题是你不想要在知道口味之后立即制作蛋糕，而是在正确的时间之后再制作。

为了解决这个额问题，你能写一个 prepareCake 函数存储你的口味。然后，在 prepareCake 中 return makeCake 函数。

使用这个方法，你能够在任何时候调用 return 的函数，蛋糕会在一秒钟之后制作。

function prepareCake (flavor) {
  return function () {
    setTimeout(_ => console.log(`Made a ${flavor} cake!`), 1000)
  }
}

const makeCakeLater1 = prepareCake('banana')
const makeCakeLater2 = prepareCake('strawberry')
const makeCakeLater3 = prepareCake('apple')

// And later in your code...
makeCakeLater1() // Made a banana cake!
makeCakeLater2() // Made a strawberry cake!
makeCakeLater3() // Made a apple cake!

这就是闭包被用来减少副作用的——在你想要的时候通过创建一个函数来激活内部的闭包

# 闭包中的私有变量

在函数内部创建的变量不能够被外部的函数访问。正因为他们不能被外部函数访问，所以称之为私有变量。

# 深拷贝与浅拷贝区别

浅拷贝：增加了一个指针指向已存在的内存地址，仅仅是指向被复制的内存地址，如果原地址发生改变，那么浅拷贝出来的对象也会相应的改变。

深拷贝：增加了一个指针并且申请了一个新的内存，使得增加的指针指向这个新的内存。因为是新的内存地址存放着复制的对象，所以原地址发生改变，深拷贝出来的对象是不会改变的。

# 函数节流、函数防抖

函数节流是减少函数的触发频率；函数防抖是延迟函数执行，且不管触发多少次都只执行最后一次

函数节流：单位时间n秒内，第一次触发函数执行 之后不管触发多少次都不执行。到下一个单位时间n秒时 第一次触发函数执行，多用于页面scroll滚动、窗口resize、防止按钮重复点击

//函数节流是减少函数的触发频率，而函数防抖则是延迟函数执行，并且不管触发多少次都只执行最后一次
// /*函数节流*/ /*单位时间n秒内，第一次触发函数并执行，以后 n秒内不管触发多少次，都不执行。直到下一个单位时间n秒，第一次触发函数并执行，这个n秒内不管函数多少次都不执行。用处：多用于页面scroll滚动，或者窗口resize，或者防止按钮重复点击等情况 */
function throttle(fn, interval) {
  var enterTime = 0;//触发的时间
  var gapTime = interval || 300 ;//间隔时间，如果interval不传，则默认300ms
  return function() {
    var context = this;
    var backTime = new Date();//第一次函数return即触发的时间
    if (backTime - enterTime > gapTime) {
      fn.call(context,arguments);
      enterTime = backTime;//赋值给第一次触发的时间，这样就保存了第二次触发的时间
    }
  };
}

函数防抖：延迟函数执行，多用于input框输入时，显示匹配的输入内容的情况

// /*函数防抖*/  /*延迟函数执行。即不管debounce函数触发了多久，只在最后一次触发debounce函数时，才定义setTimeout，到达间隔时间再执行 需要防抖的函数。用处：多用于 input 框 输入时，显示匹配的输入内容的情况*/
function debounce(fn, interval) {
  var timer;
  var gapTime = interval || 1000;//间隔时间，如果interval不传，则默认1000ms
  return function() {
    clearTimeout(timer);
    var context = this;
    var args = arguments;//保存此处的arguments，因为setTimeout是全局的，arguments不是防抖函数需要的。
    timer = setTimeout(function() {
      fn.call(context,args);
    }, gapTime);
  };
}

// post接口请求
function post(url, Data, succFun, failFun) {
  var header = {
    'content-type': 'application/x-www-form-urlencoded',
    // 'Content-Type': 'application/json',
    // 'Accept': 'application/json, text/javascript, */*; q=0.01'
  }
  if (wx.getStorageSync('token')) {
    header['Authorization'] = 'Bearer ' + wx.getStorageSync('token');//用户token
  }
  const requestTaskPost = wx.request({
    url: host + api[url],
    header: header,
    data: Data,//请求参数
    method: 'POST',
    success: (res) => {
      //成功返回
      if (res.statusCode == 401) { //如果需要token请求,但不存在返回未登录的code时统一弹出
        wx.hideLoading();
        wx.clearStorage();
        wx.showModal({
          title: '登录提示',
          content: '登录失效,请重新登录!',
          confirmText: '去登录',
          cancelText: '再看看',
          confirmColor: '#696DEE',
          success: (res) => {
            if (res.confirm) {
              // 跳转登录页面
              wx.reLaunch({
                url: '/pages/login/login',
              })
            } else {
              wx.reLaunch({
                url: '/pages/index/index',
              })
            }
          }
        });
        return false;
      }
      wx.hideLoading();
      succFun(res.data);
    },
    fail: (res) => {
      // 失败返回函数 如果需要对失败函数做处理的传入
      var errObj={msg:'',reason:res}
      if (res.errMsg == 'request:fail timeout') {
        toast('网络请求超时!');
        errObj.msg='网络请求超时!'
      }else{
        toast('服务器连接失败!');
        errObj.msg='服务器连接失败!'
      }
      
      if (failFun) {
        failFun(errObj);
      }
    },
    complete: (data) => {
      if (data.statusCode && data.statusCode != 401) {
        var strCode = data.statusCode == 408 && data.statusCode || (data.statusCode.toString()).substring(0, 1)
        statusCode[strCode] && toast(statusCode[strCode])

        if (failFun && statusCode[strCode]) {
          failFun({msg:statusCode[strCode],reason:data});
        }
      };
    }
  })
  
  //下面这部分代码就是取出请求头中的token
  requestTaskPost.onHeadersReceived(res => {
    res && res.header && res.header.Authorization && wx.setStorageSync('token', res.header.Authorization);
  });
}
